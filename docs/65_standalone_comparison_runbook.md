# Standalone 比較運用ランブック

## 前提

- Codex と Claude Code は **同じ task bundle** を読む
- 両者は互いの成果物を参照しない
- 比較は人間が目視で行う

## 推奨ディレクトリ

```text
okada-ai-codex/
okada-ai-claude/
```

## 各タスクの進め方

1. 対象 bundle を読む
2. `AGENTS.md` または `CLAUDE.md` を読む
3. まず acceptance criteria を満たす最小実装を作る
4. tests / runbook / env example を必要分だけ更新する
5. 人間が動かして確認する

## 目視比較チェックポイント

### 成果物
- acceptance を満たしているか
- テストが通るか
- 既存設計に自然に接続しているか
- 過剰設計や無関係な抽象化をしていないか

### ツールとしての印象
- 指示理解が速いか
- 勝手な設計変更が少ないか
- fail-safe を自然に入れるか
- 追加質問の質が良いか
- 修正依頼への追従が素直か

## 採用の考え方

- どちらか片方を全面採用してよい
- 片方を土台にして一部だけ吸収してもよい
- 両方とも不採用でもよい
- 厳密な点数化は不要

## 最低限の記録

各タスクごとに次を残す。
- どちらをより良いと感じたか
- 理由
- 使いたい断片が何か
